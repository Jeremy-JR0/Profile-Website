This was a group project with 3 other people. We had the choice to make whatever IOT project we wanted and we chose to make a Smart Plant Health Monitoring System. The Smart Plant Health Monitoring System IoT Project is designed to help users maintain plant health by automating watering and lighting based on sensor data. It uses a Raspberry Pi connected to multiple sensors (humidity, temperature, light, soil moisture, and soil pH) to monitor plant conditions. The system integrates AWS IoT Core, DynamoDB, and API Gateway to store and transmit data to a web dashboard, allowing users to view real-time plant conditions. Key features include:

  • Automated Watering based on soil moisture levels.
  • Automated Lighting based on light sensor readings.
  • Manual Controls for watering and lighting via the dashboard.
  • Camera Integration to analyze plant health (detecting color changes or spots).
  • Cloud-based Monitoring to access plant data remotely.
  • Calibration Feature for the ultrasonic sensor to ensure accurate water level detection.

    • The Retro Console Project aimed to develop a fully interactive gaming console named Scorpio using LilyGO T-Display S3 devices. The project involved designing and implementing hardware and software components, including wireless controllers, a VGA display, audio output, and multiple classic games. The system supported both single-player and two-player modes, featuring four games: Pong, Snake, Flappy Bird, and a custom PvP Maze Hunter. Key Features:

  • Wireless Controllers: Implemented using ESP-NOW for real-time communication.
  • VGA Output: 3-bit color VGA display with smooth graphics rendering.
  • Audio System: PWM-based sound output with music and sound effects.
  • Card Storage: Leaderboards for each game stored on an SD card.
  • Multiplayer Modes: Single-player and two-player options for enhanced gameplay.

    The UWA Book of Knowledge project aimed to organize and represent data from the University of Western Australia's handbook using Knowledge Representation techniques. The project utilized RDF (Resource Description Framework), SPARQL queries, SHACL constraints, and OWL ontologies to create a structured, flexible, and highly manipulable data model.
