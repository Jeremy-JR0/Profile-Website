Smart Plant Health Monitoring

A comprehensive IoT-based plant care system designed to monitor and maintain optimal plant health through automation and user interaction. Built around a Raspberry Pi, the system collects real-time environmental data using multiple sensors and performs automated actions like watering and lighting. A custom-built web dashboard provides live insights, historical trends, and manual control options, making the system suitable for both busy plant owners and remote monitoring.

I co-developed this project alongside three other team members. My primary responsibilities included establishing cloud connectivity using AWS services, such as configuring AWS IoT Core, setting up AWS Lambda functions, managing data storage with DynamoDB, and integrating API Gateway for communication between the frontend and backend. I also handled much of the sensor integration and programming, as well as designing and implementing the layout for the web dashboard.

System Features

  • Sensor Monitoring: Continuously tracks soil moisture, temperature, humidity, sunlight, pH levels, and water reservoir level using multiple sensors.
  • Automated Plant Care:
    • Activates a water pump when soil moisture is below plant-specific thresholds
    • Turns on LED light if sunlight levels are insufficient
  • Manual Controls:
    • Users can manually trigger the pump and grow light from the dashboard
    • A "Calibrate Water Level" button sends a custom MQTT payload to adjust the base water level reference
  • Camera Integration: Captures plant images and checks for signs of distress (e.g., color changes or spots), with plans to identify the plant type and adjust thresholds accordingly
  • Cloud Connectivity:
    • Uses AWS IoT Core with MQTT for secure bi-directional communication
    • AWS Lambda functions handle requests/responses between the device and cloud
    • DynamoDB stores both live sensor data and historical logs
  • Web Dashboard:
    • Displays current sensor data, comparison with recommended values, and visual graphs
    • Offers user-friendly controls and feedback for real-time interaction

  • Cloud and Networking - AWS IoT Core, AWS Lambda, DynamoDB, API Gateway

Retro Console

A fully custom-built retro-style game console using LilyGO T-Display S3 microcontrollers and ESP-NOW for low-latency wireless multiplayer support. The system mimics the feel of classic consoles, booting into a select screen where users can choose between multiple games, including single- and multiplayer options. One LilyGO device serves as the main console connected to a display, while others act as wireless controllers. Game files are loaded dynamically from an SD card, and high scores are saved to individual leaderboard files.

I co-developed this project with a team of three other members. My key contributions included establishing the wireless communication between the console and two controllers using ESP-NOW. Moving Target Defence

This capstone project is a web-based simulation tool designed to help users understand and visualize Moving Target Defence (MTD) techniques in cybersecurity. Built on top of the existing MTDSimTime framework, the application allows users to configure, run, and analyze simulated attacks and defenses across up to five graphs simultaneously. The simulator is fully containerized with Docker, making it easy to deploy locally or remotely. Users can customize parameters, save and submit multiple simulation scenarios, and view results such as compromised nodes, attack techniques, and defense actions. The frontend provides interactive graph visualizations and statistical breakdowns for each simulation. The application includes support for **advanced configuration** and has been structured for future expansion with minimal backend adjustments. My contributions included collaborative development across both frontend and backend components, Docker containerization for deployment, and documentation of system usage and simulation workflows.

Skills

  • Python
  • HTML
  • CSS
  • Vue.js
  • Docker

UWA Book of Knowledge

The UWA Book of Knowledge is a semantic web project focused on structuring and querying academic data from the UWA Handbook using knowledge representation techniques. Developed in collaboration with one other team member, the project uses RDF, SPARQL, SHACL, and OWL ontologies to represent, validate, and reason about relationships between university units and majors. As I co-develeped this project my main responsibilities were, designing and implementing the RDF structure and building the SPARQL query system. The project enables users to run both dynamic and static queries on the handbook data through a custom-built CLI interface.

  • SPARQL
  • SHACL
  • Owl Ontologies

Image Filter Application

This ongoing project is an image filter application built using Python's Tkinter library for the GUI, allowing users to load and apply various filters to images. It leverages foundational concepts in computer vision to implement effects such as low-pass (blurring), high-pass (sharpening), and edge detection using Sobel, Canny, and Laplacian of Gaussian filters. Future plans include extending support for video and GIF editing, making it a versatile tool for media processing.

  • Python
  • Library Manipulation: Tkinter, OpenCV, NumPy, Pillow (PIL)
